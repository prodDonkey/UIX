## 4. 风险与阻塞点
1. ADB 设备不稳定导致执行失败：准备设备健康检查接口与重试策略。
2. LLM 输出 YAML 质量波动：加强提示词约束 + 生成后强校验。
3. Midscene 输出日志格式变动：Runner 做兼容解析，保留原始日志。
4. 报告文件路径与权限：后端只暴露白名单目录，禁用任意路径访问。

## 5. 交付物清单
1. 前端工程（Vue3）。
2. 后端工程（FastAPI + uv）。
3. 数据库迁移脚本。
4. API 文档。
5. 部署文档与运行手册。
6. 测试报告与验收记录。

## 6. 建议执行顺序
1. 先完成 M1 基建与脚本管理。
2. 再完成 M2 执行链路，确保“可执行”最早上线。
3. 第三阶段补齐 AI 生成与报告体验。
4. 最后集中做质量与验收。
